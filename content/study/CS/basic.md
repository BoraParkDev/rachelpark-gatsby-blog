## 프로그래밍 언어 & 운영체제(OS)

### ● 컴파일러(Compiler)

컴파일러란 인간과 컴퓨터 사이의 번역가로서 인간의 요구를 컴퓨터가 이해할 수 있는 언어로 바꿔서 전달해주는 역할을 한다.

컴파일(Compile)의 사전적 정의는 주어진 언어로 작성된 컴퓨터 프로그램을 다른 언어의 동등한 프로그램으로 변환하는 프로세스를 수행하는 소프트웨어이다.

일반적으로 자바, 파이선, 루비 등으로 되어있는 프로그래밍 언어를 사용해서 컴파일러에게 명령을 전달하면, 컴파일러가 그 명령을 기계어인 0과 1로 번역해서 컴퓨터에게 전달해주는 방식이다. 유사한 수준의 언어사이에서는 "트랜스파일러" 혹은 "크로스 컴파일러"라고 부르기도 한다.

ex. TypeScript에서 JavaScript로 컴파일한다.

- 저수준 : 컴퓨터 친화적 언어 - 배우기 어려움, 구체적인 명세가 필요함, 조금만 에러가 있어도 버그를 뱉어냄. 대신 낮은 사양에서도 잘 작동함

- 고수준 : 인간 친화적 언어 - 사람이 학습하기 쉬움. 유지보수 용이. 컴퓨터가 알아서 행동(판단해야)하기 때문에 작동이 느림

개발자는 이러한 컴파일러에게 문서로 일을 시키는 사람이라고 보면 되는 것이다.

### ● 컴퓨터의 구성 요소

- CPU : 컴퓨터의 머리

따로 데이터를 저장하지 않음. 데이터를 연산하거나 처리할 경우 CPU로 데이터를 보내주어야 함

이럴 경우 보조기억장치에 데이터를 보내라는 신호를 준다.

- HDD, SDD : 컴퓨터의 창고 (보조기억장치라고도 불림)

메인보드에 들어가는 부품 중 하나

CPU보다 속도가 느리다. CPU와 함께 일을 하면 CPU가 계속 기다려야 하는 상황이 발생한다.

이를 위해 "메모리"가 등장했다.

- 메모리 : CPU의 개인 작업공간

속도의 하향 평준화를 방지하고자 나온 개념. 고전적인 명칭은 "스토어(store)"이기도 하다.

메모리를 만듬으로써 CPU는 보조기억장치에 필요한 데이터를 그때그때 요청하지 않아도 되게 된다.

즉, 보조기억장치에서 메모리로 한번에 큰 데이터 뭉치를 옮겨두고, 메모리안에서만 작업을 하는 것이다.

### ● 운영체제(Operating System)

운영체제는 우리 대신 하드웨어를 관리해주는 프로그램이다.

뿐만 아니라, 응용 프로그램의 기초를 제공하며 컴퓨터 사용자와 컴퓨터 하드웨어 사이의 중개자 역할을 한다.

> An operating system is a program that manages a computer’s hardware. It also provides a basis for application programs and acts as an intermediary between the computer user and the computer hardware.
> Abraham Silberschatz

ex. 하드웨어 용량이 얼마나 있는지 확인하기 → 운영체제가 보조기억장치를 관리하고 있기 때문에 가능한 일

프로그램 설치와 실행도 운영체제가 하드웨어를 컨트롤하고, CPU와 메모리를 관리해주기떄문에 우리는 몇 번의 클릭으로 프로그램을 설치하고 실행할 수 있다.

### ● 자바 최신 버전

과거의 다양했던 운영체제로 파생된 프로그래밍 언어도 굉장히 많았다. 따라서 프로그램의 버그를 수정하거나 새로운 기능을 추가해야 할 때 번거로웠고, 같은 작업을 무수히 많이 처리해야 하는 일이 발생하기 시작했다.

고심한 자바 팀은 각 운영체제 위에 JVM(Java Virtual Machie)이라는 소프트웨어를 만들게 된다.

어떤 운영체제이건 간에 이 프로그램 위에서 자바를 실행시킬 수 있도록 하는 소프트웨어로 이 JVM 위에서 자바 언어로 만든 프로그램이 돌아갈 수 있도록 만든 것이다.

그러나, 운영체제 위에 프로그램을 올리고 그 위에 또 프로그램을 돌리는 것은 속도가 느려진다는 단점이 있다. 또한 모바일에 대한 니즈는 PC와 다르고, 모바일의 크기 특성상 당연히 PC보다 작기때문에 한계가 발생한다. 이 외에도 기타 다른 이유들로 모바일은 JVM과 같은 개념이 PC보다 상대적으로 덜하다는 것을 참고하자.

💬 [참고] 우리가 흔히 들어왔던 말들

“c언어를 배워봐. 컴퓨터를 이해하는데 좋아”

→ c언어는 자바나 파이썬처럼 고수준의 언어가 아닌, 컴퓨터 친화적 언어 즉, 저수준 언어이다.

그래서 컴퓨터가 이해하기 쉬운 코드를 쳐야 하므로 자연스레 컴퓨터에 대한 이해도가 높아질 수 밖에 없는것이다.

“자바부터 배워봐. 써먹을 곳이 많아.”

→ 자바는 운영체제에 독립적일 뿐만 아니라 안드로이드에서도 사용되므로 그 생태계가 굉장하다.

사용범위가 엄청 넓기 때문에 이렇게 말하는 것

“파이썬 부터 배워봐. 그게 쉬워”

→ 파이썬은 인간 친화적 언어인 고수준 언어이다. 따라서 인간의 입장에서는 당연히 배우기 쉬울 수 밖에 없는것이다. (동의 못해...)

- MAN : Metropolitan Area Network

### ● 요청(Requeset) & 응답(Response)

파일을 보내달라는 요청이 하나의 컴퓨터로 몰리면 어떻게 될지 상상해보자.

요청이 몰린 컴퓨터는 파일을 보내기 위해 보조기억장치(HDD,SDD) 메모리, CPU를 사용한다.

파일을 보내기 위해서는 보조기억장치의 파일을 메모리에 올리고 CPU가 그 파일을 보내야하는 과정을 겪어야 하는데, 신호가 몰리면 메모리의 공간을 다쓰거나, CPU가 처리할 수 있는 한계를 넘어버리게 된다.

그러면 컴퓨터는 개복치처럼 죽는것이다… 👾 재부팅해야 살아난다.

## ● What the Ubuntu?

![우분투](https://blog.kakaocdn.net/dn/demNTF/btqWXbeDoI9/cJxPVV3lHFhPVn6ksDXBe1/img.jpg)

우분투에 대해서 알려면 먼저 리눅스(Linux)를 알아야 한다.

리눅스는 앞서 언급한 운영체제(OS) 중 하나인데, 특이한 점이 있다. 바로 서버 프로그램을 돌리기 위한 운영체제이다.

“서버 프로그램” : 클라이언트의 요청에 응답해주는 컴퓨터, 생각하고 판단하는 프로그램으로서 24시간 365일 돌아간다. 하드웨어를 직접 컨트롤하지 않기 위해 운영체제 위에서 프로그램을 돌린다. 여기서 사용하는 운영체제가 바로 ‘리눅스’인것!

💬 근데 왜 서버프로그램을 굳이 리눅스에서 돌려?

- 공짜거든~~

리눅스는 '리누스 토발스'라는 사람이 만들었는데, 무료로 배포했다.(Are U 에인졀...?👼)

> 1991년 9월 17일 리누스 토르발스가 처음 출시한 운영 체제 커널인 리눅스 커널에 기반을 둔 오픈 소스 유닉스 계열 운영 체제 계열이다.[8] 리눅스는 일반적으로 리눅스 배포판 안에 패키지 처리된다.
> 배포판에는 리눅스 커널과 지원 시스템 소프트웨어, 그리고 라이브러리를 포함하고 있으며 이들 가운데 다수가 GNU 프로젝트에 의해 제공된다
> 위키백과

리눅스가 무료 배포된 이후, 여러 사람들이 이 운영체제를 꾸준히 발전시켜왔고, 다양한 버전의 리눅스가 만들어졌다.

우분투는 리눅스의 버전 중의 하나로 하드웨어를 관리해서 사용자가 프로그램을 사용하기 쉽게 도와주는 윈도우나 맥 OS 같은 운영체제이다.

또 다른 유명한 리눅스 버전 중 하나로는 바로 레드햇(Red Hat)이 있다.

![레드햇](https://www.hellot.net/data/photos/images/000218/20200221100923424_ZH673A5N.jpg)
레드헷 : 리눅스를 개량해서 유료로 판매하는 회사

무료 버전을 유료화 해서 사용자가 과연 많을까? 라고 생각한다면 오산!

예를 들어, 보안이 중요한 회사 혹은 금융과 관련된 회사는 안정적인 서비스가 필수적이다. 서버 장애가 발생한다면 엄청난 손실이 발생하거나 정보 노출의 위험이 있다. 이럴 때 무료 OS를 사용한다면 AS 요청을 한다거나 책임을 물을 수 없고, 또한 품질을 보장해주지 않는다.

이렇기에, 레드햇은 소프트웨어 공급은 무료로 하되, 지원은 유료로 하는 비즈니스 모델을 만들어서 제품을 팔기 시작하게 된다.

리눅스의 특징에 대해서 조금만 더 설명을 해보자면,,

윈도우나 맥은 그래픽 UI가 함께 있다. 우리가 컴퓨터를 키면 항상 보는 그런 창, 화면, 설정 옵션들이 모두 UI다.

그런데, 리눅스는 그런게 필요가 없다. 왜? 서버만을 위한 운영체제여서 그래픽 UI 따위는 중요하지 않다. 서버가 24시간 365일 내내 안.정.적으로 잘 돌아가는 데에만 관심이 있기 때문에 흔히 말하는 시커먼 창이 전부다. 어쩌면 이래서 무료인지도 모르겠다.

그래서 UI적인 요소가 없기 때문에 "명령어"로만 작동한다.

리눅스 명령어에 대해서는 추후에 정리해볼 예정이다.

### ● Summary

- 서버 : 데이터를 주는 컴퓨터. 데이터를 주기 위해서는 프로그램이 필요함 (운영체제 : 리눅스)

(ex. 로그인 처리를 하기 위해서는 사용자의 아이디가 존재하는지, 비밀번호가 일치하는지 등등을 알아보는 프로그램이 필요함)

- 운영체제 : 프로그램을 돌리기 위해서 필요한 환경. why? 하드웨어를 그냥 쓸 수는 없으니까~
